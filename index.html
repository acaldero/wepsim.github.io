The Web Elemental Processor SIMulator

Access to WepSIM
Students are able to see how values are modified for each clock cycle (at microprograming level), and for each assembly instruction (at assembly level) too
For each control signals students can explorer how it works, they could change its associated values, and they could check the associated help
WepSIM comes with several examples...

Description

Critical point of view: evaluate solutions

Students are able to compare different hardware, firmware, instruction sets, etc. and to assess what are the advantages and disadvantages

Different hardware architecure could be used in WepSIM, and for each one, several firmwares (instruction sets) can be defined. So different assembly programs can be designed, coded, tested, and compared.
Integrated vision: from hardware to system software

Students gain an integrated vision of how hardware and software interplays through microprogramming and assembly programming

Instruction set could include interrupts, exceptions, system calls, etc.
WepSIM can be used as a complementary tool while teaching Computer Structure, Computer Architecture, Operating Systems, etc.
Accesibility

Improving accesibility: WCGA 2 (AAA) + Command-line alternative

WepSIM has been improved to pass the WCAG 2.0 (Level AAA) at the Web Accessibility Checker. Along with an improved command-line version, more users can use it with less problems.
Interactive

Students can explorer how an example works, and easily check "What will happen if I change..." too

It is a very valuable complement of a good exercises book.
For example, students can activate two tri-states signals and see what happens.
Mobility

Student can use it in many devices (tablet, smartphone, etc.)

It is more flexible for students to practice.
For example, WepSIM allows the students to learn while travelling on subway, or on bus.

Install on any platform
with Google Chrome 73+
As Progressive Web Application:
1.- Once the https://wepsim.github.io/wepsim is loaded in Chrome (Android, Windows, Linux), from the top-right corner tap on the menu icon (Chrome).
2.- Move within share options until 'add to home screen' option and click on it.
3.- Then click in the 'add' option.
4.- Finally WepSIM can be launched from the home screen icon.

Getting Started

A typical session with WepSIM has the following steps:
1.- From the menu, there are different examples available
(click on the dark blue 'title' of the example you want to work with)
2.- Then you could go to the microcode editor workspace
3.- You can modify the microcode (if you wish) and microcompile it
4.- Later, you could go to the assembly editor workspace
5.- You can modify (if you wish) your assembly code
(by using the instructions defined in the previous microcode) and compile it
6.- Finally, go back to the simulator workspace, and you can execute step by step and analyze the state of the components. It is possible to work at assembly level or at microcode level
